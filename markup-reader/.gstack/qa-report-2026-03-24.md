### QA-001: File dialog unavailable in headless environment

**Severity:** INFORMATIONAL
**Category:** Functional
**Repro steps:**
1. Launch app (dev server)
2. Click "Open" button or press Cmd+O
3. Observe error overlay: "Failed to open file dialog: TypeError: Cannot read properties of undefined (reading 'invoke')"

**Evidence:** `screenshots/dialog-error.png`

**Analysis:** The Tauri dialog plugin (`@tauri-apps/plugin-dialog`) relies on native IPC (`invoke`) to communicate with the Rust backend. In a headless browser environment, the dialog's `open()` call fails before the native file picker can be shown. This is a **test environment limitation**, not an app bug. The native file picker will work correctly when a user clicks "Open" in the real macOS app.

**Fix:** No fix needed for the app. For E2E testing, use Playwright's `page.waitForEvent('dialog')` to intercept and dismiss the native file picker in headless mode.
